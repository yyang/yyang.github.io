Since I just realized git submodule is very useful and important when doing a huge project, I started to deal with submodules.   However the mechanism is not described in most of the online manuals:

1) Submodule Add
git submodule add git@github.com:yyang/repo submodule_path_here
This will log the submodule in .gitmodules and register. Once we clone a repo and do git submodule init this will also register.

2) Submodule update
git submodule update will checkout certain version into the registered path. However the version is located in .git/modules/submodule_path_here/FETCH_HEAD during initial registration, and such update command will not change version number, this command works as git checkout certain_version.

3) Update version
You can either do:
cd submodule_path_here
git checkout master
git pull
Or do:
git submodule foreach git pull origin master
This will update the version in FETCH_HEAD, and fetch latest code. This requires commit to update the repo.

4) Under submodules - it will search for the nearest git repo - so whenever there's a commit it will commit to submodule, very nice.
